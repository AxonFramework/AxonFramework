= Dead Letter Queues in Axon Framework
:navtitle: Dead Letter Queues

== Introduction

In production environments, event handling failures are inevitable.
An external API may be temporarily unavailable, a downstream service may time out, or an unexpected combination of events may trigger an unhandled edge case.

By default, event processors log errors and move on, or stop processing altogether when configured to rethrow.
Neither approach is ideal: ignoring failures loses data, while stopping blocks the entire processor.

A Dead-Letter Queue (DLQ) provides a third option: park the failed event in a persistent queue so you can decide to retry it later once the underlying problem is resolved.
In addition, because events within a sequence must be processed in order, the DLQ also enqueues all subsequent events for the same sequence — ensuring that a later event is never applied before an earlier one has succeeded.

The supported dead-letter queue implementation is the `SequencedDeadLetterQueue`, which tracks failing sequences by their sequence identifier (determined by the xref:axon-framework-reference:events:event-processors/streaming.adoc#sequential-processing[sequencing policy]).
